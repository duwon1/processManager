# ADR-0002: 에이전트 인증 — 1회용 설치 토큰 → 노드 전용 secret

- **상태**: 채택
- **관련 코드**: `AgentInstallTokenService`, `AgentRegistrationService`, `NodeService`, `WebSocketAuthInterceptor`, `AgentInstallTokenController`

## 맥락

원격 서버의 Python 에이전트가 백엔드에 **아웃바운드**로 WebSocket 연결한다.
에이전트는 브라우저 세션이 없으므로 사용자 JWT를 쓸 수 없고, 다음을 만족해야 했다.

- 설치 시 사용자의 장기 자격증명(비밀번호·장수명 토큰)을 원격 서버에 남기지 않아야 한다.
- 노드가 어느 사용자 소유인지 서버가 확신할 수 있어야 한다.
- 설치 스크립트가 잘못된/만료된 토큰으로 실행되면 아무것도 설치하지 않아야 한다.

## 결정

**2단계 자격증명**으로 분리한다.

1. **1회용 설치 토큰** (`pmi_<64hex>`, 5분, 최대 2회 연장)
   - 프로필 화면에서 발급 → 설치 명령어에 담겨 원격 서버에서 실행.
   - 설치 스크립트는 `validate`(소비 없이 검증) → `claim`(agentId에 원자적 선점) 순으로 호출해,
     실제 설치 전에 유효성을 확인하고 다른 에이전트의 재사용을 막는다.
   - 최초 WebSocket CONNECT 시 `consume`으로 소비되어 노드가 등록된다. 이후 재사용 불가.
   - DB에는 원문이 아닌 SHA-256 해시만 저장한다.

2. **노드 전용 secret** (`as_<64hex>`)
   - 등록이 완료되면 서버가 노드별 secret을 발급하고(해시만 저장), 에이전트에 전달한다.
   - 이후 재접속은 설치 토큰이 아니라 이 `agent-secret`으로 수행한다.
   - secret 비교는 상수시간 비교(`MessageDigest.isEqual`)로 타이밍 공격을 방지한다.

두 경로 모두 `WebSocketAuthInterceptor`의 STOMP CONNECT 단계에서 헤더로 처리한다.

## 결과

**장점**
- 원격 서버에 남는 장기 자격증명이 **노드 1개로 범위가 제한된** secret뿐이다.
- 설치 토큰이 짧고 1회용이라 유출되어도 창이 좁고, claim으로 도난 재사용을 막는다.
- 토큰/secret 모두 해시만 저장해 DB 유출 시 원문이 드러나지 않는다.

**단점 / 비용**
- 상태 전이(발급→검증→선점→소비→secret 발급)가 많아 흐름이 복잡하다.
- 구버전 에이전트 호환(hostname 폴백, claim TTL 등) 처리 코드가 필요하다.
- 5분 만료로 설치가 느린 환경에서는 연장 또는 재발급이 필요할 수 있다.
