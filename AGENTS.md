# 규칙

- 사용자의 의도를 먼저 파악하고, 불명확하면 작업 전에 확인한다.
- 파일이나 코드를 추가, 수정, 삭제할 때는 반드시 사용자에게 먼저 확인받는다.
- 추가, 수정, 삭제 전에는 어느 부분을 어떻게 바꿀지 구체적으로 설명하고 확인받는다.
- 코드 수정 시 해당 기능이 무엇을 하는지 반드시 주석을 단다.
- 작업 후에는 개선할 점, 하지 못한 점, 잘한 점을 반드시 피드백한다.
- 버그를 사전에 잡기 위해 가능한 범위에서 디버깅하고 검증한다.
- 작업 시 개발환경과 배포환경을 모두 고려하고, 환경별 차이점과 영향 범위를 확인한다.
- 커밋 전에는 관련 빌드/린트/컴파일 검증을 수행하고 결과를 확인한다.
- 커밋, 푸쉬, 빌드, 배포는 사용자의 요청 또는 확인이 있을 때 진행하고, 진행 후 결과를 보고한다.
- 배포 전에는 현재 커밋/브랜치/변경 파일을 확인하고, 배포 후에는 서비스 상태와 로그를 확인한다.
- 관련 없는 코드는 건드리지 않는다.
