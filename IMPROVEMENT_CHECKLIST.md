# Improvement And Feature Checklist

## Product

- [ ] 대시보드 첫 진입 화면에서 노드 상태, 최근 경고, 할 일 요약을 한눈에 보여준다.
- [ ] 노드 목록에 즐겨찾기, 정렬, 검색, 상태 필터를 추가한다.
- [ ] 노드 상세 화면에 최근 접속, 최근 명령, 최근 업데이트 상태를 표시한다.
- [ ] 팀별 노드 접근 범위를 한 화면에서 비교할 수 있게 한다.
- [ ] 사용자별 권한 변경 이력을 확인할 수 있게 한다.
- [ ] 초대받은 사용자의 온보딩 흐름을 별도로 정리한다.
- [ ] 빈 상태 화면을 상황별로 정리한다.
- [ ] 주요 작업 완료 후 다음 행동을 안내하는 짧은 상태 피드백을 추가한다.

## UI/UX

- [ ] 모바일 `/teams` 화면의 정보 밀도를 더 줄인다.
- [ ] 모바일에서 주요 버튼과 보조 버튼의 시각적 우선순위를 명확히 한다.
- [ ] 선택된 팀, 선택된 멤버, 선택된 권한 상태를 더 선명하게 표시한다.
- [ ] 카드 안에 카드가 들어가는 구조를 줄인다.
- [ ] 반복되는 alert, confirm, prompt UI를 공통 Dialog 컴포넌트로 완전히 통일한다.
- [ ] Toast 메시지 문구를 작업 결과 중심으로 정리한다.
- [ ] 테이블성 정보는 모바일에서 리스트형 요약 UI로 전환한다.
- [ ] 터미널/파일 패널의 모바일 레이아웃을 별도 최적화한다.
- [ ] 버튼 안 텍스트가 작은 화면에서 줄바꿈되거나 넘치지 않게 점검한다.
- [ ] 아이콘 버튼에는 hover/focus tooltip을 붙인다.
- [ ] 로딩, 실패, 빈 상태 UI를 Bootstrap 스타일로 통일한다.

## Agent

- [ ] 에이전트 설치 성공/실패 단계를 서버에 자세히 보고한다.
- [ ] 에이전트 설치 후 `agent_secret` 저장 여부를 서버가 확인할 수 있게 한다.
- [ ] 에이전트 업데이트 전후 버전과 실패 원인을 UI에 표시한다.
- [ ] 에이전트 업데이트를 자동/수동 모드로 나눈다.
- [ ] 에이전트 health check endpoint를 추가한다.
- [ ] 에이전트 로그 일부를 UI에서 확인할 수 있게 한다.
- [ ] 에이전트 삭제 실패 시 복구 가이드를 UI에 표시한다.
- [ ] Linux 외 Windows/macOS 에이전트 지원 가능성을 분리 설계한다.
- [ ] 서버 재부팅 후 에이전트 자동 복구 상태를 UI에 표시한다.

## Monitoring

- [ ] CPU, 메모리, 디스크 임계치 알림을 추가한다.
- [ ] 노드 오프라인 알림을 추가한다.
- [ ] 서비스 다운 알림을 추가한다.
- [ ] 프로세스 CPU/메모리 급증 알림을 추가한다.
- [ ] 일정 기간의 메트릭 히스토리를 저장한다.
- [ ] 메트릭 차트에 시간 범위 선택을 추가한다.
- [ ] 노드별 리소스 추세 요약을 추가한다.
- [ ] 디스크 파티션별 사용량을 더 명확히 표시한다.
- [ ] 네트워크 송수신량을 누적/초당 기준으로 전환할 수 있게 한다.

## Terminal And Files

- [ ] 터미널 세션 목록과 현재 활성 세션 수를 표시한다.
- [ ] 터미널 세션 강제 종료 기능을 추가한다.
- [ ] 터미널 명령 실행 전 위험 명령 확인 절차를 추가한다.
- [ ] 파일 브라우저에 검색을 추가한다.
- [ ] 파일 다운로드 기능을 추가한다.
- [ ] 파일 업로드 기능을 추가한다.
- [ ] 텍스트 파일 미리보기를 추가한다.
- [ ] 이미지 파일 미리보기를 추가한다.
- [ ] 파일 권한/소유자 정보를 표시한다.
- [ ] 숨김 파일 표시 토글을 추가한다.

## Teams

- [ ] 팀 이름 변경 UX를 더 명확하게 만든다.
- [ ] 팀 삭제 전 영향받는 노드와 멤버를 보여준다.
- [ ] 팀원 권한 프리셋을 더 직관적인 이름으로 정리한다.
- [ ] 팀원별 권한 복사 기능을 추가한다.
- [ ] 팀별 기본 권한 프리셋을 설정할 수 있게 한다.
- [ ] 초대 재전송 기능을 추가한다.
- [ ] 초대 취소 기능의 상태 표시를 더 명확히 한다.
- [ ] 팀 멤버 검색과 상태 필터를 추가한다.

## Backend

- [ ] Flyway 또는 Liquibase로 DB 마이그레이션을 이전한다.
- [ ] 서비스 계층 테스트를 추가한다.
- [ ] WebSocket 권한 검증 테스트를 추가한다.
- [ ] 설치 토큰 생성/연장/소비 테스트를 추가한다.
- [ ] 팀 권한 변경 테스트를 추가한다.
- [ ] API 응답 DTO와 에러 응답 형식을 문서화한다.
- [ ] 중복되는 현재 사용자 조회 로직을 공통화한다.
- [ ] WebSocket destination 문자열을 더 강하게 타입화한다.
- [ ] 운영 로그 레벨과 개발 로그 레벨을 분리한다.
- [ ] 느린 DB 쿼리 추적 방법을 추가한다.

## Deployment

- [ ] 운영/개발 DB 설정이 섞이지 않게 프로필을 명확히 분리한다.
- [ ] Fly secrets 목록을 문서화한다.
- [ ] GitHub Actions 배포 전 검증 단계를 더 명확히 한다.
- [ ] 배포 후 health check를 자동화한다.
- [ ] 배포 실패 시 rollback 절차를 문서화한다.
- [ ] 프론트 빌드 산출물 크기 추이를 확인한다.
- [ ] Docker image size를 줄인다.
- [ ] 운영 로그 조회 방법을 문서화한다.
- [ ] 로컬 개발 서버 실행 절차를 간단히 정리한다.
